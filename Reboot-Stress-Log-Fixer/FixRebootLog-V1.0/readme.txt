# 重启压力测试日志时间修正工具（PowerShell）

本工具用于修正重启压力测试生成的日志文件中的时间不连续问题，支持**整体平移后缀**（保持前面测试时间不变）和**单点偏移**，并可预览修改效果，最终生成修正后的日志文件。

## 功能特点

- **整体平移后缀**：从指定的测试编号开始，将该测试及其之后的所有测试整体提前或延后指定时间（分钟），前面的测试时间保持不变。
- **单点偏移**：对单个测试进行偏移（实际上也是从该测试开始的后缀平移，便于快速操作）。
- **预览效果**：在修改前可查看前3个、后3个测试的时间变化，也可查看任意指定测试的详细信息。
- **重置偏移**：随时清除所有偏移，恢复原始时间。
- **自动生成输出文件**：默认在原文件名后添加 `_fixed` 后缀，不覆盖原文件。
- **支持自定义输入/输出文件**：可通过命令行参数指定。

## 适用场景

- 日志中某些测试的起始时间有偏差，需要整体调整后续所有测试的时间。
- 需要保持前面部分测试的时间不变，只修正后续时间。
- 需要快速预览偏移效果，确认无误后再写入文件。

## 环境要求

- Windows 操作系统（支持 PowerShell 5.0 或更高版本）
- 日志文件格式符合以下示例（包含 `Start Date and Time:` 和 `End Date and Time:` 行）：

```
Start Reboot Stress Test: 
Current Test 1/1000 Start Date and Time:2026-01-02 15:38:52.056 
...
Current Test 1/1000 End Date and Time:2026-01-02 15:40:24.513 
...
```

## 使用前准备

1. 下载或保存脚本文件，例如 `FixRebootLog_Final.ps1`。
2. 将脚本与需要处理的日志文件放在同一目录下（或指定完整路径）。
3. 确保 PowerShell 执行策略允许运行脚本（详见下方“常见问题”）。

直接运行（使用默认文件名）

在 PowerShell 中进入脚本所在目录，执行：

.\FixRebootLog_Final.ps1

脚本会默认读取当前目录下的 `26#_2da287ac.log`，输出文件为 `26#_2da287ac_fixed.log`。

## 交互菜单说明

运行后，脚本会解析日志并显示基本信息，然后进入交互菜单：

```
请选择操作：
  1) 修改单个测试的起始时间（提前/后退）
  2) 从某测试开始整体平移后缀（保持前面不变）
  3) 重置所有偏移
  4) 查看指定测试详情
  5) 确认并写入文件
  6) 退出（不保存）
```

### 选项 2：整体平移后缀（最常用）

1. 输入起始测试编号（如 `998`）。
2. 输入偏移分钟数（正数向后推，负数向前提，如 `60` 表示向后推1小时）。
3. 脚本会自动显示从该测试开始的 5 个测试的新时间，供确认。
4. 如需继续调整，可重复执行选项 2 或选项 1；如满意，选择 5 保存。

### 选项 1：修改单个测试的起始时间

1. 输入测试编号。
2. 输入偏移分钟数。
3. 该测试及之后的所有测试都会相应偏移（与选项 2 效果相同，只是起始编号不同）。

### 选项 3：重置所有偏移

清除所有已设置的偏移，恢复到原始时间（仅内存中，不影响已保存的文件）。

### 选项 4：查看指定测试详情

输入测试编号，查看该测试的原始时间、新时间及偏移秒数。

### 选项 5：确认并写入文件

将当前偏移应用到日志，生成新文件。生成后会在控制台显示文件完整路径。

### 选项 6：退出

不保存任何更改，退出脚本。

## 参数说明

脚本支持以下命令行参数（可选）：

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `-InputFile` | 输入日志文件名（支持相对或绝对路径） | `26#_2da287ac.log` |
| `-OutputFile` | 输出文件名（支持相对或绝对路径） | 原文件名 + `_fixed.log` |
| `-GapSeconds` | 相邻测试间隔（仅用于连续化预览，不影响实际偏移） | `0.1` |

示例：

```powershell
.\FixRebootLog_Final.ps1 -InputFile "C:\logs\test.log" -OutputFile "C:\logs\test_fixed.log" -GapSeconds 0.2
```

## 常见问题

### Q1：运行脚本时提示“无法加载文件，因为在此系统上禁止运行脚本”

**原因**：PowerShell 默认执行策略为 Restricted。

**解决方法**：在 PowerShell 中执行以下命令（仅对当前窗口生效）：

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
```

然后再运行脚本。

### Q2：脚本提示“未找到任何测试对”

**原因**：日志格式不符合预期，可能缺少 `Start Date and Time:` 或 `End Date and Time:` 行，或格式有误。

**解决方法**：检查日志文件，确保时间行格式为 `yyyy-MM-dd HH:mm:ss.fff`，且 Start 和 End 成对出现。

### Q3：偏移后预览没有变化

**原因**：可能未正确输入偏移分钟数（如输入了带文字的数字），或起始测试编号超出范围。

**解决方法**：只输入数字，如 `60` 或 `-30`。起始编号应在 1 到总测试数之间。

### Q4：输入 5 后没有生成文件

**原因**：可能没有写入权限，或输出路径不存在。

**解决方法**：将脚本和日志放在桌面或用户文档目录下，或使用管理员身份运行 PowerShell，或通过 `-OutputFile` 指定一个有权限的路径（如 `$env:USERPROFILE\Desktop\fixed.log`）。

### Q5：偏移后生成的文件中时间顺序错乱

**原因**：脚本的偏移逻辑是正确的，但如果之前已经应用过偏移，可能会累积。请先选择选项 3 重置，再重新偏移。

### Q6：只想将某个测试之后的所有测试整体平移，但前面的时间保持不变，为什么预览中前面的时间也变了？

**原因**：默认预览只显示前3个和后3个，可能你修改的测试编号较大（如 998），预览中看不到该测试的变化，但前面的时间应保持不变。你可以选择选项 4 查看指定测试详情来确认。

## 注意事项

- 本工具不会修改原日志文件，始终生成新文件。
- 偏移量单位是“分钟”，支持小数（如 0.5 分钟 = 30 秒）。
- 偏移后，测试之间的相对间隔保持不变（即前一个测试的结束到后一个测试的开始之间的时长不变）。
- 如果偏移量过大，可能会导致日期跨天，脚本会自动处理。
- 建议在正式使用前先用少量测试验证效果。

## 示例

假设需要将第 998 个测试及之后的所有测试整体向后推 1 小时，保持前面测试时间不变。

1. 运行脚本：`.\FixRebootLog_Final.ps1`
2. 输入 `2`
3. 输入起始测试编号 `998`
4. 输入偏移分钟数 `60`
5. 查看预览效果，确认无误后输入 `5` 保存
6. 生成的文件名为 `26#_2da287ac_fixed.log`

```
├── 26#_2da287ac.log             # 原始日志（示例）
└── 26#_2da287ac_fixed.log       # 修正后的日志（自动生成）
```
