# 重启压力测试日志时间修正工具

## 简介

本工具用于修正重启压力测试生成的日志文件中的时间不连续问题。支持**整体平移后缀**、**按绝对时间对齐**、**异常检测**、**撤销操作**、**导出CSV报告**，并可设置输出文件的创建/修改/访问时间。

## 主要功能

- **整体平移后缀**：从指定测试开始，将其及其后所有测试整体提前或延后。
- **按绝对时间对齐**：直接设置某个测试的绝对开始时间，自动计算偏移。
- **异常检测**：自动检测时间倒流、间隔过大、持续时间异常（基于本文件统计）。
- **撤销操作**：支持回退最近一次偏移修改。
- **导出CSV报告**：记录每个测试的原始时间、新时间、偏移量、是否修改。
- **文件时间设置**：可自定义输出文件的创建时间、修改时间、访问时间。
- **多文件处理**：一次可处理多个日志文件，每个文件独立分析。

## 环境要求

- Windows 操作系统
- PowerShell 5.0 或更高版本
- 日志文件格式包含 `Start Date and Time:` 和 `End Date and Time:` 行（时间格式为 `yyyy-MM-dd HH:mm:ss.fff`）

## 使用方法

### 1. 下载脚本

将脚本保存为 `FixRebootLog_Multi.ps1`（编码：UTF-8 with BOM）。

### 2. 运行方式

#### 方式一：交互式选择文件

在 PowerShell 中进入脚本所在目录，执行：

```powershell
.\FixRebootLog_Multi.ps1
```

按提示输入文件路径（支持通配符，如 `*.log` 或 `"a.log b.log"`）。

#### 方式二：命令行指定文件

```powershell
.\FixRebootLog_Multi.ps1 -InputFiles "test1.log","test2.log"
```

```powershell
.\FixRebootLog_Multi.ps1 -InputFiles "*.log" -OutputDir "D:\output"
```

#### 方式三：直接运行（不带参数）

默认会询问文件路径。

## 参数说明

| 参数 | 说明 | 示例 |
|------|------|------|
| `-InputFiles` | 输入日志文件（支持多个，可用通配符） | `"*.log"` 或 `"a.log","b.log"` |
| `-OutputDir` | 输出文件存放目录（默认为输入文件所在目录） | `"D:\fixed_logs"` |
| `-CreationTime` | 设置输出文件的创建时间（格式：`yyyy-MM-dd HH:mm:ss`） | `"2026-01-04 01:15:41"` |
| `-ModifyTime` | 设置输出文件的修改时间 | `"2026-01-04 01:15:41"` |
| `-AccessTime` | 设置输出文件的访问时间 | `"2026-01-04 01:15:41"` |

**注意**：时间参数可选，如果不提供，脚本会在保存后询问是否设置。

## 交互菜单说明

处理每个文件时，会进入交互菜单：

```
请选择操作：
  1) 修改单个测试的起始时间（提前/后退）
  2) 从某测试开始整体平移后缀（保持前面不变）
  3) 重置所有偏移
  4) 查看指定测试详情
  5) 确认并写入文件
  6) 跳过此文件（不保存）
  7) 按绝对时间对齐（指定测试的绝对开始时间）
  8) 撤销上一次偏移操作
```

- 偏移量单位：**分钟**（支持小数，如 `0.5` = 30秒，`-60` = 提前1小时）
- 绝对时间格式：`yyyy-MM-dd HH:mm:ss.fff`（例如 `2026-01-02 15:38:52.056`）

## 输出说明

- **修正后的日志**：文件名在原文件名后添加 `_fixed`，例如 `原文件_fixed.log`。
- **CSV报告**（可选）：与日志文件同目录，文件名相同但扩展名为 `.csv`，包含每个测试的原始时间、新时间、偏移量、是否修改。
- **文件时间属性**（可选）：可设置创建时间、修改时间、访问时间。

## 注意事项

1. **原始文件不会被覆盖**，所有修改保存到新文件。
2. **异常检测**基于当前文件的统计值（均值、标准差），间隔过大阈值 = 均值 + 3×标准差；持续时间异常阈值 = 均值 ± 3×标准差。这些阈值仅在控制台显示，不写入文件。
3. **偏移操作会记录历史**，可撤销最近一次操作。
4. **建议先在一个文件上测试**，确认效果后再批量处理。
5. 如果文件较大（数万行），处理时间可能稍长。

## 常见问题

**Q：脚本运行提示“访问被拒绝”**  
A：检查输入的路径是否是一个文件而不是文件夹，或者是否有读取权限。

**Q：生成的文件时间没有变化**  
A：确保在交互菜单中选择5保存，且偏移量正确（非0）。可用选项4查看指定测试详情确认。

**Q：如何处理多个文件**  
A：使用 `-InputFiles "*.log"` 或列出所有文件名。

**Q：能否跳过异常检测？**  
A：异常检测仅用于提示，不影响修正操作。你可以忽略异常直接进行偏移。

## 示例

```powershell
# 处理当前目录所有log文件，输出到桌面
.\FixRebootLog_Multi.ps1 -InputFiles "*.log" -OutputDir "$env:USERPROFILE\Desktop"
```

```powershell
# 处理两个指定文件，并设置修改时间
.\FixRebootLog_Multi.ps1 -InputFiles "26#_2da287ac.log","test2.log" -ModifyTime "2026-01-04 01:15:41"
```
